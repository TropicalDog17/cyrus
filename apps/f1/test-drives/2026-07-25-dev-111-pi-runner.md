# Test Drive: DEV-111 Pi runner

**Date:** 2026-07-25
**Tester:** Cyrus
**Objective:** Validate that an issue can select the Pi runner, execute through
the EdgeWorker pipeline, publish activity, and stop cleanly.

## Setup

- Server port: `3600`
- Temporary repository: `/tmp/cyrus-pi-f1-UsFUTf/repo`
- Runner selection: `CYRUS_DEFAULT_RUNNER=pi`
- Harness mode: `CYRUS_PI_MOCK=1`
- Pi package smoke checks: pinned CLI reported version `0.82.0`; a real
  offline JSONL RPC process returned a durable `sessionId` and model state from
  `get_state`
- Server:
  `CYRUS_PORT=3600 CYRUS_REPO_PATH=/tmp/cyrus-pi-f1-UsFUTf/repo CYRUS_DEFAULT_RUNNER=pi CYRUS_PI_MOCK=1 node apps/f1/dist/server.js`

## Result: PASS

- [x] F1 server health and status endpoints responded.
- [x] `[agent=pi]` routed `DEF-2` to the Pi runner despite a competing `pi`
  label.
- [x] EdgeWorker created the issue worktree and started the Pi session.
- [x] The session published routing/model thoughts and a final response through
  the normal activity sink.
- [x] Activity pagination returned the expected single-item page.
- [x] The Pi session completed successfully and accepted a stop event.

The completed session contained four activities:

1. The issue request.
2. The routing decision.
3. `Using model: pi/default`.
4. Final response: `Pi mock session completed`.

## Issues found

- The F1 scaffold could not create its initial commit until a repository-local
  test identity was configured. This is test-environment setup, not a Pi runner
  defect.
- Pi core does not expose native MCP server configuration. Cyrus maps built-in
  tool permissions and preserves custom/MCP-style names for Pi extensions, but
  an extension must make those tools available to Pi.

## Not covered

- A paid-provider model response. The deterministic mock exercised the real
  F1/EdgeWorker/session/activity pipeline; separate mapper tests replay Pi's RPC
  event shapes, and the pinned real CLI's `get_state` command was smoke-checked.
- Pi-extension installation and an extension-provided MCP bridge.

## Conclusion

The Pi runner is wired through Cyrus's real routing and session pipeline. The
mocked provider boundary kept the test deterministic while still validating the
integration seams owned by Cyrus.
